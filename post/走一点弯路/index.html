走一点弯路——建博客也不是那么容易哇

Posted on Tue, Dec 31, 2024
阅读 | 共1406字，阅读约 3 分钟

光看教程觉得建基于Hugo的博客还是很简单的嘛毕竟只有几行代码，但实际做的时候发现还是很容易踩坑的。但幸好今天值班💔，可以有一个晚上的空闲时间来钻研一下这个东西。

Hugo本体的安装其实非常简单，MacOs 的终端和brew让安装Hugo变成执行一行代码的过程，所以翻车原因不在这里。

翻车部分主要在于怎样把Hugo和Github pages关联起来。
这部分主要参考了以下几个教程：
1.利用Hugo+Github Pages搭建个人博客 - Serendiplay
2.使用 Github、Hugo 搭建个人博客_hugo教程

在搭建完成以后回过头复盘，其实可以看出来整个逻辑挺简单的。 即使用Hugo生成一个静态网页，把包含这个网页信息的public文件夹通过git语法远程上传到github上已经搭建好的repo里。

但各教程中指令稍有冲突（或者说指令模糊），最开始试图把它们大杂烩起来但很显然这样做并不明智。于是推库（？）重建，再多次尝试以后终于成功了。 姑且放上一些我踩到的坑。

Mistake1：忘了把本地的git库和远程库连起来。

通过删库重新输代码解决，很笨但有效。

Mistake2：分支名错误（git pull命令报错）

当前分支没有跟踪信息。
请指定您要合并哪一个分支。
详见 git-pull(1)。

    git pull <remote> <branch>

如果您想要为此分支创建跟踪信息，您可以执行：

    git branch --set-upstream-to=origin/<branch> master

不知道为什么实际生成的分支是origin/main,而所有提供的教程里远程上传用的都是main。 最后把main追踪到origin/main解决。多谢提示！

git branch –set-upstream-to=origin/main main

Mistake3：远程上传拒绝请求

出现过以下两种情况

error: 无法推送一些引用到 ‘git@github.com:xxx’
提示：更新被拒绝，因为远程版本库包含您本地尚不存在的提交。这通常是因为另外
提示：一个版本库已推送了相同的引用。再次推送前，您可能需要先合并远程变更
提示：（如 ‘git pull’）。
提示：详见 ‘git push –help’ 中的 ‘Note about fast-forwards’ 小节。

提示：更新被拒绝，因为您当前分支的最新提交落后于其对应的远程分支。
提示：再次推送前，先与远程变更合并（如 'git pull ...'）。详见
提示：'git push --help' 中的 'Note about fast-forwards' 小节

本质就是远程和本地库的不同步。但网上的解决方案大多都是先用fetch再用merge指令，但我一merge就提示无效，大概是又绕回到mistake2上。解决了2，3也就解决了。

当然最开始还遇上了（因为是新电脑）所以没有SSH权限导致拒绝的情况，即

git@github.com: Permission denied (publickey)

参考教程解决。

Mistake4：搞混了yaml和toml

unmarshal failed: toml: expected character =

出现这个提示就是因为标题原因，参考hugo论坛贴解决。 +++是toml —是yaml 不能用混啊啊啊啊！！

finally

总算出现静态网页，总算可以用二级域名直接登录博客，总算能够写出第一篇属于这个博客的博文！大喜报！！
当然装修过程中因为看不懂theme提供的可选项反复尝试最后也不知道哪个对应哪个，所以这个界面我其实没有特别满意来着，之后再倒腾吧！！
另外还有一些使用github action达到自动发布文章功能，网页搜索功能，二级网页（？）功能还没实现，感觉也是需要折腾很久的东西……今天暂时没有精力搞了还是就这样算了吧。
重拾markdown语法，感觉自己好像没有全忘光好耶。
